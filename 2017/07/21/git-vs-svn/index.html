git对比svn

  • git
  • svn

  svn与git对比操作记录。

操作\vcs git svn
克隆(检出) git clone svn checkout –username=
查看版本库信息 × svn info
查看版本库下的文件 × svn list
查看状态 git status svn status
添加文件 git add(stage暂存(多次)) svn add(版本管理器追踪(一次))
删除文件 git rm svn rm
提交 git commit -m “message” svn commit -m “message”
远程提交 git push ×
删除远程分支 git push : ×
查看日志 git log –pretty=oneline svn log
diff git diff svn diff
使用远程仓库更新本地 git pull svn update
恢复未提交的 git checkout – svn revert /-R
恢复已提交的 git reset –hard HEAD^/cmid, HEAD指向当前commit的指针, ^上一个commit svn merge -r : (不能删除历史记录，只能合并到老的版本然后做一次新的提交)
命令的历史 git reflog(想要获取比当前版本更新的cmid) ×
冲突解决 git pull然后修改再提交 1.svn up更新选择df, 2.选择p推迟冲突, 3.修改文件, 4.svn resolve –accept=working 通知解决
标签 git tag -a [-m “message”][cmid] svn copy –revision= tags/
创建分支 git checkout -b svn copy trunk branches/
删除分支 git checkout -d

疑问：

  1. git pull和git fetch的区别：前者不光是拉数据还会和本地分支合并。以下两者等价：
    git pull == git fetch && git merge origin/master, 默认分支为master的情况下。
  2. git checkout 不用-b也可以创建分支了？git checkout -b == git branch
    && git checkout 。
  3. git有暂存区的概念，是追踪修改，而不是追踪文件。svn只需要一次add，其余的直接commit即可。
